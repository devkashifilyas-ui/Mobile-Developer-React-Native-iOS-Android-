services/apiClient.ts
import axios from 'axios';
import * as SecureStore from 'expo-secure-store';

export const apiClient = axios.create({
  baseURL: process.env.EXPO_PUBLIC_API_URL,
  timeout: 10000,
});

// Attach JWT from SecureStore on every request
apiClient.interceptors.request.use(async (config) => {
  const token = await SecureStore.getItemAsync('jwt_token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

JWT-based auth with NestJS backend. Tokens stored in expo-secure-store (encrypted native keychain). Refresh handled transparently via Axios interceptor.

mobile-architecture / api-integration
NestJS API Integration
Axios client layer with interceptors for JWT attachment, token refresh, and retry logic. React Query handles caching, background refetch, and loading states.
api/interceptors.ts — retry + refresh
// Auto-refresh on 401, retry once
apiClient.interceptors.response.use(
  (res) => res,
  async (error) => {
    if (error.response?.status === 401) {
      const newToken = await refreshToken();
      if (newToken) return apiClient(error.config);
      logout(); // hard logout if refresh fails
    }
    return Promise.reject(error);
  }
);
hooks/useCachedQuery.ts
// React Query wrapper for NestJS endpoints
export function useUserProfile(userId: string) {
  return useQuery({
    queryKey: ['user', userId],
    queryFn: () => apiClient.get(`/users/${userId}`),
    staleTime: 5 * 60 * 1000, // 5 min cache
    retry: 2,
  });
}

mobile-architecture / release-management
Release Management
Semantic versioning, OTA updates for JS-only changes (no store review), and store releases for native changes. Both platforms managed from a single workflow.
🔢
Expo Versioning
runtimeVersion in app.json controls native binary version. autoIncrement in eas.json handles buildNumber/versionCode automatically on each EAS build.
⚡
OTA Updates
JS/asset-only changes ship via expo-updates without waiting for App Store review. Users get updates silently in background. Critical hotfixes in minutes, not days.
📦
Store Releases
Native changes require a full EAS build + store submission. eas submit handles TestFlight and Play Internal track upload automatically.
mobile-architecture / performance
Performance Optimization
Production mobile apps must be fast on low-end Android and older iOS devices. These are the strategies applied across the codebase.
🦥
Lazy Loading
React Navigation lazy loads screens. Heavy components wrapped in React.lazy() with Suspense. Images loaded with expo-image (faster than Image).
♻
Memoization
React.memo on list item components. useMemo for expensive computations. useCallback for stable event handlers passed to FlatList.
🗄️
API Caching
React Query with staleTime prevents redundant requests. Optimistic updates for mutations. Offline-first queries persist to AsyncStorage.
🔄
Background Tasks
expo-background-fetch for data sync. Heavy processing moved off the JS thread with react-native-reanimated for animations.
